# Internationalization (i18n)

This site uses **[next-intl](https://next-intl.dev)** with the App Router. URLs always include the locale: `/en/...` and `/fr/...`.

## Where things live

| Piece | Purpose |
|--------|---------|
| `i18n/routing.ts` | Supported locales (`en`, `fr`) and `localePrefix: "always"`. **Add a new locale here** and add `messages/<code>/`. |
| `i18n/request.ts` | Loads merged message modules per request. |
| `i18n/navigation.ts` | **Use `Link`, `redirect`, `useRouter`, `usePathname` from here** — not `next/link` — so paths stay locale-aware. |
| `middleware.ts` | Resolves locale and redirects `/` → default locale. |
| `messages/<locale>/` | JSON namespaces: `common`, `home`, `pricing`, `metadata`. |

## Adding or editing copy

1. Open the right file under `messages/en/` and mirror the key in `messages/fr/` (or the other way around).
2. Prefer **one namespace per page or concern** (`home`, `pricing`, `common`) so files stay small.
3. For lists or tier tables, store **arrays/objects in JSON** and use `getTranslations(...).raw('path.to.key')` in server components or `useTranslations().raw()` in client components.
4. **Internal links**: import `Link` from `@/i18n/navigation` and use paths **without** the locale prefix (e.g. `href="/pricing"`).

## Server vs client

- **Server components** (default): `import { getTranslations } from 'next-intl/server'`, then `const t = await getTranslations('home');` and `t('hero.title')`.
- **Client components** (`'use client'`): `import { useTranslations } from 'next-intl'`, then `const t = useTranslations('home');`.

## Language switch

`components/language-switcher.tsx` keeps the current path and switches `locale` via `Link` from `@/i18n/navigation`.

## HTML `lang`

`components/locale-html-attributes.tsx` sets `document.documentElement.lang` from the active locale on the client so the root layout can stay a single shell.
